Glossary

시스템 설계 문서에서 반복되는 C# 타입·인터페이스와 Core, Stat, Effect, Skill, Combat, Status 용어, UML 기초 문법을 빠르게 찾는다.

표기 기준 · C# Core Stat Effect Skill UML

EntityId

C# 객체 참조 대신 Entity를 안정적으로 참조하는 값 타입 식별자다. 보통 readonly struct로 표현한다.

EntityRef

EntityId와 EntityRegistry 해석 정책을 함께 다루는 참조 값이다.

Source identity

Modifier, Effect, Status가 어디서 왔는지 나타내는 원천 식별 개념이다. 공개 C# 계약에서는 별도 문자열 SourceId 대신 구조화된 SourceRef를 사용한다. JavaScript 실습이나 로그의 sourceId는 SourceRef.InstanceId가 있으면 그 값을, instance가 없는 System source라면 DefinitionId를 평탄화한 검색용 보조 필드일 뿐 독립 계약이 아니다.

SourceRef

readonly record struct로 표현하는 불변 C# 값 타입이며 Kind와 DefinitionId를 가진다. SkillExecution과 Status는 InstanceId가 필수이고 System은 생략할 수 있다. 공개 예제의 ID는 command.fireball.cast.0001처럼 점 구분 문법을 사용한다.

TagSet

fire, spell, boss, debuff, dot처럼 규칙 분기와 필터링에 쓰는 태그 묶음이다.

GameContext

ActorId, TargetId, Position, Source, Tags, Time, RandomStream처럼 실행에 필요한 값을 묶어 설명하는 개념적 umbrella다. 현재 C# 참조에는 이 이름의 타입이 없으며 StatContext·EffectContext와 SkillRequest가 각 경계에 필요한 값만 소유한다.

EventBus

시스템 간 후속 반응을 느슨하게 연결하는 이벤트 발행/구독 장치다.

IGameClock

Duration, Tick, Cooldown, Schedule 같은 시간 기준을 제공하도록 권장하는 Normative conceptual contract다. Core 페이지의 C# 모양은 adapter 경계를 설명하지만 현재 source/csharp/ 프로젝트에는 이 인터페이스 선언이나 구현이 없다.

RandomStream

seed 기반 랜덤 스트림이다. 전투 판정 재현과 리플레이를 가능하게 한다.

DataRegistry

StatDefinition, EffectDefinition, SkillDefinition 같은 정적 데이터를 조회하고 검증하는 저장소다.

DebugTrace

Stat 계산, Skill 실행, Effect 결과, Combat 로그를 하나의 원인 체인으로 묶는 추적 정보다.

StatDefinition

스탯 종류의 개념적 정적 설계 데이터다. 표시명, 기본값, 최소/최대, 표시 형식, 반올림 정책을 포함하며 현재 C# 실행 참조에는 이 타입이 없다.

StatInstance

특정 Entity가 가진 스탯 하나의 개념적 목표 런타임 상태다. 현재 C# 실행 참조에는 이 타입이 없다.

StatModifier

스탯 최종값에 영향을 주는 변경 단위다. Operation과 구조화된 SourceRef를 가진다.

StatContext

조건부 Modifier 평가에 필요한 문맥이다.

CalculationPipeline

Modifier를 정해진 순서로 적용해 최종값을 계산하는 파이프라인이다.

EffectDefinition

효과의 개념적 정적 설계 데이터다. Type, TargetSelector, ValueExpression, Conditions, Tags를 가지며 현재 C# 실행 참조에는 이 타입이 없다.

EffectBundle

Effects, Reactions, ExecutionPolicy를 함께 묶은 실행 단위다. EffectBundleResult는 주 transaction의 commit 여부, 적용 operation 수, queue에 넣은 reaction 수를 보존한다. post-commit reaction의 실제 성공·실패는 이후 command의 결과와 trace에서 별도로 확인한다.

EffectContext

Effect 실행 중 필요한 CasterId, InitialTargetId, Source, RandomSeed를 담는 최소 C# 실행 문맥이다.

TargetSelector

Effect가 적용될 대상 후보를 찾고 필터링한다.

ValueExpression

회복이나 자원 변화처럼 Effect가 직접 소유하는 값 계산식이다. DamageEffect는 FormulaId, BaseValue, CoefficientBps를 DamageRequest에 담고 피해 계산은 CombatFormula에 위임한다.

EffectExecutor

Effect type별 실제 실행자다.

EffectResult

호출 분기에 필요한 최소 성공·commit 정보만 담는다. 대상별 계산, 실패 근거, 이벤트와 디버그 메시지는 EffectTrace와 각 소유 시스템 결과에 별도로 둔다.

SkillDefinition

스킬의 개념적 정적 설계 데이터다. Costs, Cooldown, Targeting, Tags, Timeline, EffectBundle 참조를 가지며 현재 C# 실행 참조에는 이 타입이 없다.

SkillRuntime

특정 Entity가 가진 스킬의 장기 availability 상태다. 배운 레벨, 쿨다운, 차지, runtime override를 보관하며 한 번의 시전 상태는 SkillExecution에 둔다.

SkillExecution

한 번의 cast 또는 channel을 나타내는 임시 실행 인스턴스다. ExecutionId, Phase, ReservedCost, TargetSnapshot, CancellationToken 같은 실행 생명주기를 소유한다.

SkillBook

Entity가 배운 스킬 목록, 레벨, 해금 상태, 장착 슬롯을 관리한다.

SkillRequest

C# 스킬 사용 요청이다. CasterId, SkillId, TargetId, RequestedTick, RootSeed를 포함한다.

SkillValidator

비용, 쿨다운, 상태 이상, 무기 조건, 거리, 시야, 대상 유효성을 검증한다.

CostPolicy

스킬 비용을 언제 지불하고 언제 환불하는지 정의한다.

CooldownPolicy

쿨다운 시작 시점, 그룹 쿨다운, 차지 회복, 쿨다운 감소 적용 방식을 정의한다.

TargetingSpec

self, explicit target, ground position, direction, area preview 같은 입력 규칙을 정의한다.

ActionTimeline

windup, cast, release marker, active frame, recovery, cancel window를 정의한다.

SkillResult

스킬 사용 결과. 성공 여부, 실패 이유, 실행된 EffectResult 목록을 담는다. 계산 근거는 DebugTrace, commit된 사실은 outbox event로 분리한다.

DamageRequest

DamageEffect가 ICombatResolver에 전달하는 C# 피해 계산 요청이다.

DamageResult

Combat Resolution이 반환하는 불변 계산 결과다. HitOutcome, Critical, RawDamage, ResolvedDamage, ShieldAbsorbed, FinalHpDamage, Overkill을 포함하며 ResolvedDamage = ShieldAbsorbed + FinalHpDamage + Overkill을 만족한다. 아직 상태 변경이 commit됐다는 사실은 아니다.

BPS · Basis Points

비율을 정수 고정소수점으로 표현하는 단위다. 10,000 BPS = 100%, 1 BPS = 0.01%이며, 곱셈 뒤 어느 단계에서 어떤 방식으로 반올림하는지까지 NumericPolicy에 포함해야 한다.

StatusDefinition

버프/디버프의 개념적 정적 설계 데이터다. Duration, StackPolicy, TickPolicy, Modifiers, RemoveConditions를 정의하며 현재 C# 실행 참조에는 이 타입이 없다.

StatusInstance

특정 Entity에게 적용된 상태의 런타임 상태다.

Idempotency · 멱등성

같은 commandId나 idempotency key로 요청을 반복해도 상태 변경과 이벤트를 한 번만 확정하는 성질이다.

ReactionQueue

commit된 event에서 만든 상태 변경 ReactionCommand를 결정론적 순서로 실행하는 queue다. 한 번의 Drain은 dispatch 중 추가된 반응까지 같은 causation wave로 묶고 depth·총 개수·총 budget·idempotency 상한을 적용한다. 상한 초과나 handler 예외는 명시적으로 실패하며 아직 dispatch하지 않은 반응을 다음 drain으로 넘기지 않는다. UI·로그 같은 관찰 subscriber는 이 queue와 분리한다.

ClockDomain

지속시간·쿨다운·tick이 어느 시간축을 따르는지 지정하는 계약이다. 예를 들어 simulation_tick과 real-time 계열은 정지·배속·재현 의미가 다르므로 서로 직접 비교하지 않는다. 선택한 domain과 현재 tick은 replay envelope에 함께 고정한다.

StopPolicy

주 operation 후보를 상태 변경 전에 어떤 단위로 plan에 포함할지 정하는 정책이다. all_or_nothing, best_effort_per_target, stop_on_first_success처럼 선택 경계를 나타내며, 이미 commit된 작업을 되감거나 commit 이후 reaction 실패를 처리하는 정책은 아니다.

Outbox

상태 변경과 함께 원자적으로 저장하는 이벤트 발행 기록이다. commit 성공 뒤 별도 전달자가 이 기록을 외부로 발행한다.

Precondition · Optimistic Version

Resolve가 읽은 entity version을 expectedVersion으로 기록하고 commit 직전에 다시 비교하는 낙관적 동시성 조건이다.

Context Fingerprint

Modifier가 실제로 읽는 target·tag·거리 같은 문맥 값을 정규화해 만든 캐시 식별자다. 값이 다르면 같은 Stat 질의라도 캐시를 분리한다.

stableOrderKey

우선순위가 같은 reaction이나 event의 순서를 매 실행에서 같게 만드는 안정 정렬 키다. 생성 시각이나 컬렉션 순회 순서에 의존하지 않는다.

Replay · Version Envelope

입력 snapshot과 seed뿐 아니라 runtime·RNG·수치·definition·formula 버전을 함께 고정하는 재현 계약이다.

Invariant · 불변 조건

리소스가 음수가 아니거나 피해량 합계가 보존되는 것처럼, 정상 상태의 commit 전후에 항상 참이어야 하는 규칙이다.

CorrelationId · CausationId

CorrelationId는 Fireball 한 번처럼 전체 실행 흐름을 묶고, CausationId는 현재 command나 event를 직접 만든 직전 원인을 가리킨다.

Runtime Snapshot

Resolve가 읽는 특정 시점의 불변 상태다. 계산에 필요한 entity 값과 version을 포함하며 원본 상태를 직접 변경하지 않는다.

Atomic Commit · Rollback

모든 operation과 invariant를 working copy에서 검증한 뒤 상태와 outbox를 한 번에 확정하고, 하나라도 실패하면 전부 폐기하는 처리다.

SkillMutation

SkillDefinition 원본을 수정하지 않고 비용, 쿨다운, 태그, Effect를 변경하는 런타임 오버라이드다.

readonly struct

EntityId처럼 값 의미와 불변성이 필요한 작은 C# 타입에 사용한다. 여러 필드를 묶는 원천 정보는 SourceRef 같은 별도 값 타입으로 표현한다.

interface

IStatQuery, ICombatResolver처럼 소비자가 필요한 동작만 노출하는 C# 계약이다.

불변 요청·결과

SkillRequest와 DamageResult는 생성자에서 값을 확정하고 읽기 전용 속성으로 노출한다.

IReadOnlyList / IReadOnlyDictionary

서비스 밖에는 읽기 전용 컬렉션을 노출하고, 생성자에서 입력 배열을 복사해 외부 변경이 내부 상태에 반영되지 않게 한다.

아키텍처 다이어그램 최소 문법

표기 의미 읽는 법
사각형 클래스 박스 데이터 또는 책임을 가진 객체 SkillDefinition은 비용/쿨다운/타임라인 데이터를 가진다.
실선 / 실선 화살표 연관 관계 / 탐색 가능한 방향 클래스 다이어그램의 실선은 연관을, 시퀀스·흐름도의 화살표는 호출 또는 이동 방향을 뜻한다.
점선 화살표 Dependency 의존 관계 한 요소가 다른 요소의 타입이나 계약을 사용하며 변경 영향을 받을 수 있음을 뜻한다.
빈 삼각형 화살표 실선은 Generalization, 점선은 Realization 클래스 상속은 실선+빈 삼각형, 인터페이스 구현은 점선+빈 삼각형으로 구분한다.
빈 / 채운 다이아몬드 Aggregation / Composition 빈 다이아몬드는 약한 집합, 채운 다이아몬드는 생명주기를 함께 소유하는 강한 합성을 뜻한다.
상태 다이어그램 런타임 상태 변화 Ready → Casting → Cooldown처럼 생명주기를 읽는다.

Combat 용어

용어 설명
DamageRequest DamageEffect가 ICombatResolver에 전달하는 피해 계산 요청이다. AttackerId, DefenderId, Source, FormulaId, DamageType, BaseValue, CoefficientBps, Tags, Seed를 포함한다.
DamageResult ICombatResolver가 반환하는 불변 계산 결과다. Outcome은 bool Hit가 아니라 HitOutcome(Hit, Miss, Blocked, Immune, Rejected)이며 Critical, RawDamage, ResolvedDamage, ShieldAbsorbed, FinalHpDamage, Overkill을 포함한다. ResolvedDamage = ShieldAbsorbed + FinalHpDamage + Overkill 보존식을 만족하고, 아직 상태 변경 사실은 아니다.
BPS 10,000을 100%로 보는 정수 비율 단위다. 예를 들어 2,500 BPS는 25%이며, 반올림 위치와 방식은 formula·numeric policy의 일부다.
DamageCommitted DamageResult가 상태 저장소의 atomic commit에 성공한 뒤 발행되는 도메인 사실이다. IReactionQueue는 이 이벤트부터 상태 변경 반응을 만들 수 있다.
CombatContext 현재 C# 실행 참조에서는 이미 해석된 ScalingStatValue, Outcome, Critical, ResistanceBps, 가용 Shield·HP를 담는 계산 fixture다. JavaScript Runtime Lab은 keyed RNG로 Hit/Crit를 판정한다. 거리·지형·상태 tag와 방어·관통 resolver를 결합하는 문맥은 두 참조 모두의 개념 확장이다.
HitResolver 명중, 회피, 블록, 무적, 회피 프레임을 판단하는 구성 요소.
CriticalResolver 치명타 확률, 치명타 배율, 치명타 저항을 계산하는 구성 요소.
DamageFormula 기본 피해, 스탯 계수, 공격자 보정으로 공식 중간값을 만든다. 방어, 저항, 보호막, 반올림은 DamagePipeline의 별도 단계가 담당한다.
Resistance fire, ice, poison, physical 같은 피해 타입별 감소율.
Penetration 대상의 저항이나 방어 일부를 무시하는 공격자 속성.
Shield / Barrier / Absorb HP 피해 전에 피해를 흡수하거나 전환하는 방어 계층.
CombatLog 피해 계산 근거를 사람이 읽을 수 있게 기록한 로그.

Status 용어

용어 설명
StatusDefinition 상태의 정적 데이터다. Tags, Duration, Stacking, Tick, Modifiers, PeriodicEffects를 정의한다.
StatusInstance 특정 Entity에게 적용된 상태의 런타임 데이터다. InstanceId, SourceRef, ExpiresAt, StackCount, NextTickAt을 가진다. Modifier는 SourceRef.Status(StatusDefinitionId, StatusInstanceId)를 원천으로 등록하고 제거할 때 같은 구조화된 값을 사용한다.
StackPolicy 상태 재적용 시 refresh, add stack, replace, independent stack 중 무엇을 할지 정하는 규칙.
DurationPolicy 지속시간을 갱신, 연장, 고정, 감소, 만료하는 규칙.
TickPolicy DoT, HoT, aura처럼 주기적으로 실행되는 타이밍 규칙.
Dispel / Cleanse 특정 태그나 타입의 상태를 제거하는 규칙.
Immunity 상태 적용 자체를 거부하는 방어 규칙.
DoT / HoT Damage over Time / Heal over Time. Status tick에서 periodic Effect로 실행하는 것이 안전하다.
Snapshot 상태 적용 시점의 스탯을 저장해 tick마다 같은 기준으로 계산하는 방식.
Dynamic Recalculation tick마다 현재 스탯과 상태를 다시 읽어 계산하는 방식.
