Glossary

시스템 설계 문서에서 반복되는 Core, Stat, Effect, Skill, Combat, Status 용어와 UML 기초 문법을 빠르게 찾는 레퍼런스다.

Core Stat Effect Skill UML

EntityId

런타임 객체 포인터 대신 Entity를 안정적으로 참조하는 식별자다.

EntityRef

EntityId와 EntityRegistry 해석 정책을 함께 다루는 참조 값이다.

SourceId

Modifier, Effect, Status, Item, Reward가 어디서 왔는지 추적하는 원천 식별자다.

SourceRef

kind, definitionId, instanceId를 함께 가진 구조화된 원천 참조다. 공개 예제의 ID는 command.fireball.cast.0001처럼 점 구분 문법을 사용한다.

TagSet

fire, spell, boss, debuff, dot처럼 규칙 분기와 필터링에 쓰는 태그 묶음이다.

GameContext

actor, target, position, source, tags, time, random stream을 담는 실행 문맥이다.

EventBus

시스템 간 후속 반응을 느슨하게 연결하는 이벤트 발행/구독 장치다.

TimeService

duration, tick, cooldown, schedule 같은 시간 기준을 제공한다.

RandomStream

seed 기반 랜덤 스트림이다. 전투 판정 재현과 리플레이를 가능하게 한다.

DataRegistry

StatDefinition, EffectDefinition, SkillDefinition 같은 정적 데이터를 조회하고 검증하는 저장소다.

DebugTrace

Stat 계산, Skill 실행, Effect 결과, Combat 로그를 하나의 원인 체인으로 묶는 추적 정보다.

StatDefinition

스탯 종류의 정의 데이터. 표시명, 기본값, 최소/최대, 표시 형식, 반올림 정책을 포함한다.

StatInstance

특정 Entity가 가진 스탯 하나의 런타임 상태다.

StatModifier

스탯 최종값에 영향을 주는 변경 단위다. operation과 sourceId를 가진다.

StatContext

조건부 Modifier 평가에 필요한 문맥이다.

CalculationPipeline

Modifier를 정해진 순서로 적용해 최종값을 계산하는 파이프라인이다.

EffectDefinition

효과의 정적 데이터. type, targetSelector, valueExpression, condition, tags를 가진다.

EffectBundle

여러 EffectDefinition을 순서 또는 정책에 따라 묶은 실행 단위다.

EffectContext

Effect 실행 중 필요한 source, target, position, tags, random, stat lookup 정보를 담는다.

TargetSelector

Effect가 적용될 대상 후보를 찾고 필터링한다.

ValueExpression

회복이나 자원 변화처럼 Effect가 직접 소유하는 값 계산식이다. DamageEffect는 formulaId와 scaling 입력만 DamageRequest에 담고 피해 계산은 CombatFormula에 위임한다.

EffectExecutor

Effect type별 실제 실행자다.

EffectResult

Effect 실행 성공 여부, 대상별 결과, 실패 이유, 이벤트, 디버그 로그를 담는다.

SkillDefinition

스킬의 정적 데이터. 비용, 쿨다운, 타겟팅, 태그, 타임라인, EffectBundle 참조를 가진다.

SkillRuntime

특정 Entity가 가진 스킬의 현재 상태. 쿨다운, 차지, 시전 중 여부를 보관한다.

SkillBook

Entity가 배운 스킬 목록, 레벨, 해금 상태, 장착 슬롯을 관리한다.

SkillRequest

스킬 사용 요청. caster, skillId, target, aimPosition, inputTime, seed를 포함한다.

SkillValidator

비용, 쿨다운, 상태 이상, 무기 조건, 거리, 시야, 대상 유효성을 검증한다.

CostPolicy

스킬 비용을 언제 지불하고 언제 환불하는지 정의한다.

CooldownPolicy

쿨다운 시작 시점, 그룹 쿨다운, 차지 회복, 쿨다운 감소 적용 방식을 정의한다.

TargetingSpec

self, explicit target, ground position, direction, area preview 같은 입력 규칙을 정의한다.

ActionTimeline

windup, cast, release marker, active frame, recovery, cancel window를 정의한다.

SkillResult

스킬 사용 결과. 성공/실패, 실패 이유, 실행된 EffectResult, 이벤트, 디버그 로그를 포함한다.

DamageRequest

DamageEffect가 Combat Resolution에 전달하는 피해 계산 요청이다.

DamageResult

Combat Resolution이 반환하는 최종 피해 계산 결과다.

StatusDefinition

버프/디버프의 정적 데이터. duration, stack, tick, modifier, remove condition을 정의한다.

StatusInstance

특정 Entity에게 적용된 상태의 런타임 상태다.

SkillMutation

SkillDefinition 원본을 수정하지 않고 비용, 쿨다운, 태그, Effect를 변경하는 런타임 오버라이드다.

아키텍처 다이어그램 최소 문법

표기 의미 읽는 법
사각형 클래스 박스 데이터 또는 책임을 가진 객체 SkillDefinition은 비용/쿨다운/타임라인 데이터를 가진다.
실선 / 실선 화살표 연관 관계 / 탐색 가능한 방향 클래스 다이어그램의 실선은 연관을, 시퀀스·흐름도의 화살표는 호출 또는 이동 방향을 뜻한다.
점선 화살표 Dependency 의존 관계 한 요소가 다른 요소의 타입이나 계약을 사용하며 변경 영향을 받을 수 있음을 뜻한다.
빈 삼각형 화살표 실선은 Generalization, 점선은 Realization 클래스 상속은 실선+빈 삼각형, 인터페이스 구현은 점선+빈 삼각형으로 구분한다.
빈 / 채운 다이아몬드 Aggregation / Composition 빈 다이아몬드는 약한 집합, 채운 다이아몬드는 생명주기를 함께 소유하는 강한 합성을 뜻한다.
상태 다이어그램 런타임 상태 변화 Ready → Casting → Cooldown처럼 생명주기를 읽는다.

Combat 용어

용어 설명
DamageRequest DamageEffect가 CombatResolver에 전달하는 피해 계산 요청. attacker, defender, SourceRef, damageType, formulaId, baseValue, scaling 입력, tags를 포함한다.
DamageResult CombatResolver가 반환하는 불변 계산 결과. hit, critical, rawDamage, resolvedDamage, plannedHpDamage, plannedShieldAbsorb, failureReason, debugTrace를 포함하며 아직 상태 변경 사실은 아니다.
DamageCommitted DamageResult의 operation이 atomic commit에 성공한 뒤 발행되는 도메인 사실이다. ReactionQueue는 이 이벤트부터 상태 변경 반응을 만들 수 있다.
CombatContext 거리, 방향, 지형, 상태 태그, 난이도, RandomStream 같은 전투 상황 데이터.
HitResolver 명중, 회피, 블록, 무적, 회피 프레임을 판단하는 구성 요소.
CriticalResolver 치명타 확률, 치명타 배율, 치명타 저항을 계산하는 구성 요소.
DamageFormula 기본 피해, 스탯 계수, 공격자 보정으로 공식 중간값을 만든다. 방어, 저항, 보호막, 반올림은 DamagePipeline의 별도 단계가 담당한다.
Resistance fire, ice, poison, physical 같은 피해 타입별 감소율.
Penetration 대상의 저항이나 방어 일부를 무시하는 공격자 속성.
Shield / Barrier / Absorb HP 피해 전에 피해를 흡수하거나 전환하는 방어 계층.
CombatLog 피해 계산 근거를 사람이 읽을 수 있게 기록한 로그.

Status 용어

용어 설명
StatusDefinition 상태의 정적 데이터. tags, duration, stacking, tick, modifiers, periodicEffects를 정의한다.
StatusInstance 특정 Entity에게 적용된 상태의 런타임 데이터. instanceId, appliedBySourceId, remainingTime, stackCount, nextTickTime을 가진다. Modifier 제거에는 보통 instanceId를 sourceId로 사용한다.
StackPolicy 상태 재적용 시 refresh, add stack, replace, independent stack 중 무엇을 할지 정하는 규칙.
DurationPolicy 지속시간을 갱신, 연장, 고정, 감소, 만료하는 규칙.
TickPolicy DoT, HoT, aura처럼 주기적으로 실행되는 타이밍 규칙.
Dispel / Cleanse 특정 태그나 타입의 상태를 제거하는 규칙.
Immunity 상태 적용 자체를 거부하는 방어 규칙.
DoT / HoT Damage over Time / Heal over Time. Status tick에서 periodic Effect로 실행하는 것이 안전하다.
Snapshot 상태 적용 시점의 스탯을 저장해 tick마다 같은 기준으로 계산하는 방식.
Dynamic Recalculation tick마다 현재 스탯과 상태를 다시 읽어 계산하는 방식.
