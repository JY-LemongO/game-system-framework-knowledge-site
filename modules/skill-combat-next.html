Equipment / Progression 확장안

Combat과 Status가 본문 시스템으로 추가되었으므로, 다음 확장은 장비와 성장이다. 장비는 StatModifier와 TriggeredEffect를 공급하고, Progression은 스킬 해금, 특성, 보상 선택, SkillMutation으로 기존 시스템을 바꾼다.

Equipment ItemInstance Affix SetBonus Reward SkillMutation

다음 우선순위

핵심 수직 슬라이스의 개념 구조는 갖춰졌지만 Equipment로 바로 확장하기 전에 실행 계약을 단단하게 해야 한다. Phase 3의 첫 작업은 Runtime Contract Hardening, 그 다음이 Equipment / Item이다.

Runtime Contract Gate → Equipment / Item → TriggeredEffect → Progression / Reward

Equipment / Item System

객체 책임
ItemDefinition 아이템의 정적 데이터. 타입, 슬롯, 기본 옵션, 태그를 가진다.
ItemInstance 개별 아이템 인스턴스. unique id, roll, durability, bind 상태를 가진다.
EquipmentSlot main_hand, armor, ring, relic 같은 장착 위치를 정의한다.
EquipmentController 장착, 해제, 교체, 요구 조건 검증을 담당한다.
ModifierTemplate 장비가 제공할 StatModifier의 템플릿이다.
TriggeredEffect OnAttackHit, OnKill, OnDodge 같은 이벤트에 반응해 Effect를 실행한다.
SetBonus 세트 장비 개수에 따라 Modifier 또는 Effect를 활성화한다.

Progression / Reward System

객체 책임
ProgressionState 레벨, 경험치, 선택한 보상, 해금 상태를 관리한다.
RewardDefinition 보상의 정적 데이터. StatBonus, AddSkill, SkillMutation, TriggeredEffect 등을 표현한다.
RewardChoiceSet 로그라이트식 선택지 묶음을 구성한다.
RewardApplicator 보상을 Stat, Skill, Effect, Equipment 쪽 변경으로 변환한다.
TalentNode 특성 트리 노드와 연결 조건을 표현한다.
SkillMutation 스킬의 비용, 쿨다운, 효과, 태그, 범위를 변경하는 규칙이다.

주의할 의존성

상황 잘못된 구조 권장 구조
장비 공격력 증가 장비가 최종 공격력을 직접 계산 ItemInstance sourceId로 StatModifier 등록
장비 발동 효과 장비가 Combat 내부 함수를 직접 호출 EventBus의 OnDamageResolved를 구독해 Effect 실행
레벨업 보상 Progression이 SkillDefinition 원본을 직접 수정 SkillMutation 또는 Runtime Override 적용
세트 효과 장착/해제 때 수동으로 값 더하기/빼기 SetBonus sourceId로 Modifier 등록/제거

권장 작업 범위

  • 먼저 Phase 3 Readiness의 execution identity, commit, reaction ordering, cache/tick 계약을 구현하고 회귀 테스트를 통과한다.
  • 먼저 Equipment / Item System 본문 문서를 추가한다.
  • 장비 장착/해제 시 sourceId 기반 Modifier 등록/제거 예제를 만든다.
  • TriggeredEffect가 EventBus와 EffectSystem을 통해 실행되는 흐름을 다이어그램화한다.
  • 그 다음 Progression / Reward System을 추가해 SkillMutation과 보상 선택을 연결한다.
